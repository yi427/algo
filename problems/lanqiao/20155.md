# 20155. 修改数位

## [题目描述](https://www.lanqiao.cn/problems/20155/learning/)

给定一个 n 位的没有前导零的十进制数 m，你可以将任意位 ai 改为任意其它数字 bi，花费为 |bi - ai|。我们希望通过最少的花费使得修改后的数中存在连续的 10 位，包含了从 0 到 9 的所有数字，且每个数字恰好出现一次。请输出最少需要的花费是多少（修改后也要求没有前导零）。

**输入格式：**
- 输入一行包含一个整数表示 m

**输出格式：**
- 输出一行包含一个整数表示答案

**样例输入：**
```
123456789301
```

**样例输出：**
```
1
```

**样例说明：**
将右边第 3 位改为 2 是一种方案，此时后 10 位恰好各含一个 0-9 各一个。

**数据范围：**
- 对于 40% 的评测用例，1 ≤ n ≤ 500
- 对于 60% 的评测用例，1 ≤ n ≤ 5000
- 对于所有评测用例，1 ≤ n ≤ 10⁶

## 解题思路

本题使用滑动窗口枚举所有可能的连续 10 位，然后计算将其修改为包含 0-9 各一个数字的最小代价。

**核心思想：**
- 枚举所有长度为 10 的连续子串
- 对于每个窗口，将其中的数字排序
- 将排序后的数字依次映射到 0-9，计算代价
- 取所有窗口中的最小代价

**为什么排序后映射是最优的？**
要使 10 个数字变为 0-9 各一个，最优策略是将最小的数字改为 0，第二小的改为 1，以此类推。这样可以最小化总的修改代价。

**关于前导0的处理：**

题目要求修改后不能有前导0。如果窗口从第一位开始（left = 0），直接将排序后的最小值映射到 0 会导致前导0。

但实际上，我们可以证明：**当窗口从第一位开始时，如果需要避免前导0，可以通过调整映射而不增加代价。**

**证明：**

设 s[0] 是第一位的原始值（s[0] ≠ 0，因为原数没有前导0），w 是窗口中排序后的某个值。

考虑两种映射方案：
- 方案1：将 s[0] 映射到 0，将 w 映射到 1
- 方案2：将 s[0] 映射到 1，将 w 映射到 0

方案1的代价：|s[0] - 0| + |w - 1| = s[0] + |w - 1|

方案2的代价：|s[0] - 1| + |w - 0| = |s[0] - 1| + w

当 s[0] ≥ 1 且 w ≥ 1 时：
- 方案1：s[0] + (w - 1) = s[0] + w - 1
- 方案2：(s[0] - 1) + w = s[0] + w - 1

**两种方案代价相等！**

这意味着：即使窗口从第一位开始，我们也可以通过交换映射来避免前导0，而不会增加额外代价。因此，代码中不需要特殊处理前导0的情况。

## 代码实现

```python
def solve(s: str) -> int:
    res = float('inf')
    for left in range(len(s) - 10 + 1):
        window = s[left:left+10]
        cost = 0
        for i, c in enumerate(sorted(window)):
            cost += abs(int(c) - i)
        res = min(res, cost)
    return int(res)


if __name__ == '__main__':
    m = input()
    print(solve(m))
```

**说明：**
- 枚举所有长度为 10 的窗口
- 对窗口内的字符排序
- 计算将排序后的数字依次映射到 0-9 的代价
- 取所有窗口中的最小代价

## 复杂度分析

- **时间复杂度**：O(n × 10 log 10) = O(n)，其中 n 是字符串长度，每个窗口排序需要 O(10 log 10)
- **空间复杂度**：O(1)，只使用常数额外空间
