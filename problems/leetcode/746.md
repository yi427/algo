# 746. 使用最小花费爬楼梯

## [题目描述](https://leetcode.cn/classic/problems/min-cost-climbing-stairs/description/)

给你一个整数数组 `costs`，其中 `costs[i]` 是从楼梯第 `i` 个台阶向上爬需要支付的费用。

一旦你支付此费用，即可选择向上爬一个或者两个台阶。你可以从下标为 `0` 或下标为 `1` 的台阶开始爬楼梯。

请你计算并返回达到楼梯顶部的最低花费。

**示例 1：**
- 输入：`costs = [10,15,20]`
- 输出：`15`

**示例 2：**
- 输入：`costs = [1,100,1,1,1,100,1,1,100,1]`
- 输出：`6`

**提示：**
- `2 <= costs.length <= 1000`
- `0 <= costs[i] <= 999`

## 解题思路

这道题可以用记忆化递归解决。

设 `dfs(i)` 表示到达第 `i` 个台阶时的最小花费，那么转移来自两种可能：
- 从第 `i - 1` 个台阶走一步过来
- 从第 `i - 2` 个台阶走两步过来

所以有：

```text
dfs(i) = min(dfs(i - 1), dfs(i - 2)) + costs[i]
```

边界处理：
- `i < 0` 时返回 `0`
- 最终楼顶不对应数组下标，因此答案是 `min(dfs(n - 1), dfs(n - 2))`

## 代码实现

### 解法一：记忆化递归

```python
from functools import cache


class Solution:
    def minCostClimbingStairs(self, costs: List[int]) -> int:
        @cache
        def dfs(i: int) -> int:
            if i < 0:
                return 0

            return min(dfs(i - 1), dfs(i - 2)) + costs[i]

        n = len(costs)
        return min(dfs(n - 1), dfs(n - 2))
```

**说明：**
- `dfs(i)` 表示到达下标 `i` 的最小花费
- 使用 `cache` 避免重复计算
- 楼顶可以由最后一个或倒数第二个台阶到达

## 复杂度分析

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`，用于递归栈和缓存
